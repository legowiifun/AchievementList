## Read this to add new games/achievements to the viewer

# Adding a new game
To add a new game, you first need to create gameName.json, and put it in this folder. It can be in subfolders of this folder. If you don't know JSON, feel free to look at the examples to learn. 

In this json file, you create an array of JSON objects with ```[]```. In-between the [], you put a series of {}'s, one for each achievement set. 

These achievement sets have several properties you can configure. For "image", you store the image of the achievement set somewhere, and put the path to it, starting in the resources folder, here. For example, if it is just in the resources folder, as a neighbor of this file, you put "img.jpg". This program supports a variety of file types. For "name", you put the name of the achievement set that is displayed on the screen. Lastly, for "achievements", you put [], and then fill out the inside of that. 

Inside achievements arrays, you put a list of {}'s, one for each achievement in the set. They are separated by commas. There are several configurable properties for these achievements. For "name", you put the name of the achievement. For "description", you put the description of the achievement. For "unlocked", you put true if you have that achievement, and false if you don't. For "unlockDate", you put "" if you have not unlocked that achievement, and the date to display if you have unlocked it, in your preferred date format. Lastly, for img, you first store the achievement art somewhere in this folder, and put the path to it, starting in the resources folder, here. For example, if it is just in the resources folder, as a neighbor of this file, you put "img.jpg". This program supports a variety of file types. 

Now that you have set up all the achievement sets and achievements, now you need to edit games.json. Before the closing `]` character, put a comma at the end of the last `}`, and on the next line, start a new {}. In this, put various configurable properties. For "name", put the name of the game. For "img",you first store the game art somewhere in this folder, and put the path to it, starting in the resources folder, here. For example, if it is just in the resources folder, as a neighbor of this file, you put "img.jpg". This program supports a variety of file types. For "jsonName", you put the path to the game.json mentioned above. This is similar to the image path, in terms of where you start. You have to include the file extension. Lastly, for "platform", put the platform that this version of the game is for. You can have the same game on a variety of different platforms, and you can even share image files between them.
